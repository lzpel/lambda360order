# lambda360（受注システム / CPQ）: 競合・強み・営業

## 1. 競合

### 国内
- **MISUMI meviy**
  - 3D CADアップロード → 要件入力（材質/数量/表面処理/公差など）→ 即時見積・発注の導線が強い。 :contentReference[oaicite:0]{index=0}
- **CADDi Quote（キャディ）**
  - 見積業務・調達活動のDX、見積情報の活用に強い（“見積の業務プロセス”側に寄る）。 :contentReference[oaicite:1]{index=1}
- **白銅（Hakudo）「描いて見積り・注文 / DATAで見積り・注文」**
  - ブラウザ作図や3D CADアップロードから、最短1分見積・注文を訴求（素材/加工領域が刺さる顧客がいる）。 :contentReference[oaicite:2]{index=2}
- **DMM.make 3Dプリント**
  - 3Dデータアップロードで簡易見積〜発注導線（主に3Dプリント領域）。 :contentReference[oaicite:3]{index=3}

### 国外
- **Xometry Instant Quoting Engine**
  - CADアップロードで即時価格・納期＋DFMフィードバック。 :contentReference[oaicite:4]{index=4}
- **Protolabs**
  - CADアップロード→即時見積のプラットフォームを長年運用（対応フォーマットや導線が成熟）。 :contentReference[oaicite:5]{index=5}
- **Hubs（Protolabs Network）**
  - CADアップロード→即時見積、仕様変更のリアルタイム反映を明示。 :contentReference[oaicite:6]{index=6}
- **SendCutSend**
  - DXF/STEP等アップロードでインスタントプライシング（板金・レーザー等に強い）。 :contentReference[oaicite:7]{index=7}
- **JLC3DP（海外）**
  - 3Dファイルアップロード→即時見積の導線を前面に出している。 :contentReference[oaicite:8]{index=8}


## 2. lambda360受注システムの「強み（売り）」の作り方
> “meviy / Xometry型”の強さは「見積が速い」だけでなく、「仕様が確定する」「手戻りが減る」「購買が回る」まで含めた体験設計にある。
lambda360はここを **“業者ごとのルールを実装できる”** 方向で差別化すると勝ち筋が立ちやすい。

### 強みA: パラメトリック発注（CAD不要〜CAD最小化）
- 歯数・モジュール・圧力角・バックラッシ…のような**パラメータ入力**で仕様確定
- CADができない発注者でも「注文できる」＝受注母数が増える
- “型番/規格品”と“セミカスタム”の間を埋める（ここが空白になりがち）

### 強みB: 制約チェック & DFM（発注前に事故を潰す）
- パラメータ制約（干渉、最小肉厚、加工可能範囲、工具制約、材料×処理の可否）
- NG時は理由と修正候補（例：最小R、穴ピッチ、ねじ下穴など）を返す
- 結果として、メール往復・電話確認・手戻り工数を削減

### 強みC: “業者の見積ロジック”をそのまま実装できる
- 見積が属人的な業者ほど刺さる（係数テーブル、工程分解、外注費、検査費、急ぎ係数）
- 「PDF見積」「注文請書」「検査成績書要否」まで一気通貫
- 既存の業務フロー（Excel見積、基幹/ERP）と併用しやすいI/Fを用意する

### 強みD: ビュワー統合（lambda360view）で“伝達ミス”を減らす
- 3Dプレビュー、寸法/注記、面粗さ・公差・処理の指定を視覚化
- 「何を発注したか」が関係者全員に同じように見える（購買・設計・現場・サプライヤ）

### 強みE: “埋め込み可能”を前提にする（業者のサイトに載る）
- 業者サイトに埋め込める（見積フォーム + 3D表示 + 発注）→営業導線が短い
- 受注側は自社ブランドで運用できる（マーケットプレイス化を強制しない）


## 3. 営業方法（最短で案件を取る順）

### フェーズ1: 1業種1社の「設計パートナー」を獲得（PoC）
- 対象は **受注単価が高い/見積が面倒/仕様がパラメトリックに落ちる** 領域
  - 例：歯車、板金定番形状、治具プレート、シャフト、スペーサ、ブラケット等
- 提案は「Webフォーム作ります」ではなく  
  **“見積工数を月◯時間削減し、取りこぼし問い合わせを受注に変える”**
- 成果物：実運用の見積PDF、受注データCSV、管理画面の最低限

### フェーズ2: “埋め込みウィジェット”として横展開
- 業者サイトに「自動見積・発注」導線を追加（問い合わせ→図面待ち→見積待ちの摩擦を削る）
- SEOが効くキーワード（例：「歯車 発注 歯数」「板金 見積 即日」）で流入を取りに行ける

### フェーズ3: 購買側SaaSではなく「サプライヤー側の武器」として売る
- 受注側（加工屋/メーカー）にとっての価値：
  - 見積担当のボトルネック解消
  - 小口・短納期の取りこぼし防止
  - 仕様不備の削減（クレーム/作り直しコスト減）

### アウトバウンド（実務）
- 既存の加工業者に刺さる1枚資料：
  - 現状：見積の往復、図面不備、納期回答の遅れ
  - 提案：Webで仕様確定→自動見積PDF→発注
  - KPI：見積リードタイム、失注率、担当工数、問い合わせ→受注率
- デモは「業者の実案件」をその場で流し込み、見積PDFが出るところまで見せる

### インバウンド（積み上げ）
- “パラメトリック例”のギャラリーを作る（ギア/ブラケット/治具等）
- 「パラメータ入力 → 3D表示 → 見積PDF」までの動画を短く作る
- 技術ブログ：制約処理/DFM/公差の扱い（検索流入が強い）


## 4. ポジショニングの結論（刺さる言い方）
- meviy / Xometryが「CADアップロード即見積」なら、
- lambda360は **“パラメトリックで仕様を確定させ、業者ルールで即見積・受注まで完結”**
- つまり **CPQ + DFM + 埋め込み受注導線** が売り。
