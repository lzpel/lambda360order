# examples 整備フロー設計

## 現状の課題

- `examples/` が `widget/` の中に埋まっており、frontend から直接参照できない
- `frontend/app/page.tsx` の例一覧は unpkg への外部リンクのみ（ローカル開発中に確認不可）
- ローカルで例を確認するには widget をビルドして dist を経由する必要がある
- 同期先が `widget/dist/` のみ

## 目標

- `examples/` を `frontend/examples/` へ移動し、frontend から直接参照できるようにする
- `make run`（frontend の Next.js dev server）で例一覧をすぐ確認できる
- `make generate` 1回で `frontend/examples/` と `widget/dist/examples/` の両方に同期
- 例の追加・修正サイクルを高速化

## 提案アーキテクチャ

```
lambda360form/
├── widget/
│   ├── src/
│   │   └── examples.ts              ← generateDemos.ts から改名。examplesDir を ../frontend/examples/ に変更、出力先を2箇所に
│   └── dist/
│       └── examples/                ← 同期先② unpkg 向け（現状維持）
└── frontend/
    ├── examples/                    ← widget/examples/ から移動（ソース兼同期先①）
    │   ├── 01_板金制御盤ボックス/
    │   │   ├── script.ts
    │   │   ├── index.html           ← examples.ts が生成
    │   │   └── README.md
    │   ├── ...
    │   ├── list.json                ← examples.ts が生成
    │   ├── demo.html.template
    │   └── index.html.template
    │   # generateDemo.ts は削除（widget/src/examples.ts に統合済みのため不要）
    └── app/
        └── page.tsx                 ← list.json を読んで例一覧リンクを表示
```

## 変更内容

### 1. widget/examples/ を frontend/examples/ に移動

```sh
git mv widget/examples frontend/examples
```

### 2. widget/src/generateDemos.ts → widget/src/examples.ts に改名・変更

ファイルを改名し、`examplesDir` のパスと出力先を変更する。

```ts
// 変更前（generateDemos.ts）
const examplesDir = path.join(__dirname, "../examples");
const outDir = path.join(__dirname, "../dist/examples");

// 変更後（examples.ts）
const examplesDir = path.join(__dirname, "../../frontend/examples");  // frontend/examples/ 参照
const outDirs = [
  examplesDir,                                                         // frontend/examples/（ソースと同じ場所に HTML 生成）
  path.join(__dirname, "../dist/examples"),                            // widget/dist（unpkg 向け）
];
```

`dist/index.html` の出力先も同様に `frontend/examples/index.html` と `widget/dist/index.html` の 2 箇所へ。
`list.json` は `frontend/examples/list.json` に出力する。

### 3. widget/package.json の scripts を更新

```json
"generate": "npx tsx src/examples.ts"
```

### 4. frontend/app/page.tsx に例一覧を追加

```tsx
// frontend/app/page.tsx（静的 import）
import demoList from '../../examples/list.json';
// ...
{demoList.map(d => (
  <li key={d.name}>
    <a href={`/examples/${d.name}/index.html`} target="_blank">{d.name}</a>
    <small> — {d.description}</small>
  </li>
))}
```

Next.js で `/examples/` を静的配信するには `next.config.ts` に設定を追加するか、
`frontend/examples/` を `public/` 以下にシンボリックリンクする方法がある。

### 5. Makefile の変更

ルートの `Makefile` の generate ターゲットで widget → generate が呼ばれれば
自動的に `frontend/examples/` も更新される。追加対応不要。

## 開発フロー

```
frontend/examples/XX/script.ts を編集
    ↓
make generate           # frontend/examples/ に HTML 生成 + widget/dist/examples/ に同期
    ↓
cd frontend && make run # Next.js dev server (localhost:3000) で確認
    ↓
page.tsx のリストから /examples/XX_.../index.html を開く
    ↓
繰り返し
```

## メリット・デメリット

| | メリット | デメリット |
|---|---|---|
| **この方式** | ソースと生成物が `frontend/examples/` に集約。frontend からの参照がシンプル | make generate が必要（hot reload 不可） |
| **別案: vite dev + iframe** | リアルタイム更新 | widget の vite dev と frontend を同時起動する必要あり |

hot reload が必要になったら vite dev + concurrently を検討する。現状は `make generate && make run` サイクルで十分。
