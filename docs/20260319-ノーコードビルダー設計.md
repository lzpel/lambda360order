# ノーコードビルダー設計

## 背景と課題

Lambda360Formの技術基盤（STEP→GLBリアルタイム変換、パラメトリック3Dプレビュー）は強力だが、
顧客（板金加工業者等）がlambda関数をJavaScriptで書く必要があり、導入障壁が高い。

20個の既存exampleを分析した結果、**全てのlambdaが同じパターンで構成されている**ことがわかった。
これはコードではなく**設定データ**として表現できる。

---

## 分析：既存examplesのパターン

### 入力パターン（全20例共通）

| 種別 | 用途 | 出現率 |
|------|------|--------|
| number (range) | 寸法指定 (幅/奥行き/高さ/径/長さ) | 18/20 |
| number (enum) | 規格サイズ選択 | 6/20 |
| select | 色選択 | 19/20 |
| text (email) | 連絡先 | 1/20 |

### 形状パターン

| パターン | 構造 | 出現率 |
|----------|------|--------|
| STEPそのまま | `step` のみ | 4/20 |
| 1軸伸縮 | `stretch(step, 1軸)` | 10/20 |
| 2軸伸縮 | `stretch(step, 2軸)` | 3/20 |
| 3軸伸縮 | `stretch(step, 3軸)` | 3/20 |

### 価格計算パターン

| パターン | 例 | 出現率 |
|----------|-----|--------|
| 線形1変数 | `base + param × coeff` | 7/20 |
| 線形多変数 | `base + Σ(param_i × coeff_i)` | 8/20 |
| 非線形（体積比例）| `base + d² × L × factor` | 3/20 |
| 固定価格 | `base` のみ | 2/20 |

**結論：全exampleが「STEP + stretch + 線形/非線形価格式」の組み合わせで表現可能。**

---

## 設計方針

### コードを生成するのではなく、コードを不要にする

ビルダーは **FormConfig（JSON）** を生成し、ランタイムがそれを直接解釈して `Output[]` を返す。
lambda関数のコード生成は行わない。

```
従来:  顧客がscript.tsを書く → lambda関数 → Output[]
今後:  顧客がGUIで設定 → FormConfig(JSON) → ランタイムが解釈 → Output[]
```

上級者向けにはlambda関数による自由記述も引き続きサポートする（エスケープハッチ）。

## FormConfig スキーマ

```typescript
/** ノーコードビルダーが生成するJSON設定 */
interface FormConfig {
  version: 1;
  product: {
    name: string;           // 商品名（表示用）
    description?: string;   // 説明文
  };

  /** STEPファイル参照 */
  step: {
    content_hash: string;   // アップロード済みSTEPのハッシュ
    filename?: string;      // 元ファイル名（表示用）
  };

  /** パラメータ定義（UI入力フィールドになる） */
  params: ParamDef[];

  /** 形状変形の定義 */
  shape: ShapeDef;

  /** 価格計算の定義 */
  pricing: PricingDef;

  /** 送信ボタン設定（省略可） */
  action?: ActionDef;
}
```

### パラメータ定義

```typescript
interface ParamDef {
  key: string;              // 変数名（pricing式で参照）
  label: string;            // 表示ラベル
  type: "number" | "select" | "text";

  // number用
  unit?: string;            // "mm", "U" 等
  default?: number;
  min?: number;
  max?: number;
  step?: number;
  enum?: number[];          // 離散値リスト（min/max/stepと排他）

  // select用
  options?: { value: string; label: string }[];

  // text用
  variant?: "text" | "email" | "area";
  placeholder?: string;
}
```

### 形状定義

```typescript
interface ShapeDef {
  /** 3D表示の軸設定 */
  axisUp?: "X" | "Y" | "Z";
  axisCenter?: ("X" | "Y" | "Z")[];

  /**
   * 伸縮定義。各軸ごとに「どのパラメータで伸縮するか」を指定。
   * 省略した軸は伸縮しない。
   */
  stretch?: {
    x?: StretchAxis;
    y?: StretchAxis;
    z?: StretchAxis;
  };
}

interface StretchAxis {
  cut: number | string;     // 切断面位置。数値リテラル or 式 ("$length * 0.5")
  delta: number | string;   // 伸縮量。通常は "$param - defaultValue" の形
}
```

**設計判断**: `cut`と`delta`に式(string)を許容することで、
`cut: "$depth * 0.5"` のような動的切断面にも対応する。
GUIでは「切断面＝パラメータの50%地点」のようなプリセットで選ばせる。

### 価格定義

```typescript
interface PricingDef {
  /** 基本価格（円） */
  base: number;

  /**
   * 加算項。各項は base に加算される。
   * 例: { param: "width", coeff: 10 } → width × 10 を加算
   */
  terms: PricingTerm[];

  /** 表示フォーマット。デフォルト: "価格: ¥{price}" */
  format?: string;

  /** 価格の丸め。デフォルト: 1（端数なし） */
  round?: number;
}

type PricingTerm =
  | { type: "linear"; param: string; coeff: number }
  // ↑ param × coeff
  | { type: "product"; params: string[]; coeff: number }
  // ↑ param1 × param2 × ... × coeff （体積比例等）
  | { type: "power"; param: string; exp: number; coeff: number };
  // ↑ param^exp × coeff
```

**これで全20 exampleの価格ロジックを表現できる:**

| example | FormConfig表現 |
|---------|---------------|
| `5000 + width * 10` | `base: 5000, terms: [{ type: "linear", param: "width", coeff: 10 }]` |
| `15000 + units*800 + depth*10` | `base: 15000, terms: [linear×2]` |
| `1000 + d²×L×0.0005` | `base: 1000, terms: [{ type: "product", params: ["diameter","diameter","length"], coeff: 0.0005 }]` |

### 送信ボタン定義

```typescript
interface ActionDef {
  label: string;                // "見積もりを送信"
  email_to?: string[];          // 固定宛先
  email_to_param?: string;      // パラメータから取得する宛先（emailフィールドのkey）
  email_bcc?: string[];
  slack?: string[];             // Slack Webhook URL
  disable_unless?: string;      // 必須パラメータのkey（空なら無効化）
  disable_message?: string;     // 無効時のメッセージ
}
```

### 各ステップの詳細

#### ① STEPアップロード
- 既存の `/upload` 機能を再利用
- アップロード → BREP変換 → content_hash取得 → 3Dプレビュー表示
- **ここが起点。STEPを上げた瞬間にプレビューが出る**

#### ② パラメータ設定
- 「+ パラメータ追加」ボタンで行を追加
- 各行: `ラベル` / `型(数値|選択|テキスト)` / `制約(範囲or選択肢)` / `単位`
- テンプレートから選択も可能:
  - 「寸法（幅）」→ number, unit: mm, min: 100, max: 1000, step: 10
  - 「色選択」→ select, よくある色プリセット
  - 「メールアドレス」→ text, variant: email

#### ③ 伸縮設定（最も重要なUX課題）
- X/Y/Z各軸に対して「どのパラメータで伸縮するか」をドロップダウンで選択
- **切断面(cut)の設定**:
  - デフォルト: STEP形状のバウンディングボックス中心（自動計算）
  - 手動: スライダーで切断面位置を調整（3Dプレビューに切断面を可視化）
- **プレビューで即時確認**: パラメータを変えると3Dモデルがリアルタイムに変形

#### ④ ライブプレビュー
- 左パネルの設定を変えるたびにリアルタイム更新
- 上段: 3Dモデル（Lambda360View）
- 下段: 顧客が見るフォームUIのプレビュー（Lambda360Form）
- **「顧客として試す」モード**: 設定パネルを隠して実際の顧客体験をシミュレーション

#### ⑤ 価格設定
- 基本価格を入力
- 「+ 価格要素を追加」で加算項を追加:
  - 「パラメータ × 係数」（線形）
  - 「パラメータA × パラメータB × 係数」（面積・体積比例）
- 価格プレビューがリアルタイムで更新される
- **ガイド機能**: 「幅300mmの時の希望価格は？」「幅600mmの時は？」→ 係数を自動逆算

#### ⑥ 送信設定
- ボタンラベル
- メール通知先（固定 or パラメータから取得）
- Slack通知先
- 必須フィールド設定

## ランタイム：FormConfig → Output[]

FormConfigを受け取り、従来のlambda関数と同じ `Output[]` を返す純粋関数。
widget側に組み込む。

```typescript
function evalFormConfig(config: FormConfig, values: Record<string, any>): Output[] {
  const outputs: Output[] = [];

  // 1. Shape出力
  let shape: ShapeNode = { op: "step", content_hash: config.step.content_hash };

  if (config.shape.stretch) {
    const cut: number[] = [0, 0, 0];
    const delta: number[] = [0, 0, 0];
    const axes = ["x", "y", "z"] as const;
    axes.forEach((axis, i) => {
      const s = config.shape.stretch?.[axis];
      if (s) {
        cut[i] = evalExpr(s.cut, values);
        delta[i] = evalExpr(s.delta, values);
      }
    });
    shape = { op: "stretch", shape, cut, delta };
  }

  outputs.push({
    type: "shape",
    shape,
    ...(config.shape.axisUp && { axisUp: config.shape.axisUp }),
    ...(config.shape.axisCenter && { axisCenter: config.shape.axisCenter }),
  });

  // 2. 価格メッセージ出力
  if (config.pricing) {
    let price = config.pricing.base;
    for (const term of config.pricing.terms) {
      switch (term.type) {
        case "linear":
          price += (values[term.param] ?? 0) * term.coeff;
          break;
        case "product":
          price += term.params.reduce((acc, p) => acc * (values[p] ?? 0), term.coeff);
          break;
        case "power":
          price += Math.pow(values[term.param] ?? 0, term.exp) * term.coeff;
          break;
      }
    }
    const round = config.pricing.round ?? 1;
    price = Math.round(price / round) * round;
    const format = config.pricing.format ?? "価格: ¥{price}";
    outputs.push({
      type: "message",
      messageType: "text",
      label: format.replace("{price}", price.toLocaleString()),
    });
  }

  // 3. アクションボタン出力
  if (config.action) {
    const emailTo = [
      ...(config.action.email_to ?? []),
      ...(config.action.email_to_param ? [values[config.action.email_to_param] ?? ""] : []),
    ].filter(Boolean);
    const disabled = config.action.disable_unless
      ? !values[config.action.disable_unless]
      : false;
    outputs.push({
      type: "action",
      label: config.action.label,
      email_to: emailTo,
      email_bcc: config.action.email_bcc ?? [],
      slack: config.action.slack ?? [],
      disable: disabled ? (config.action.disable_message ?? true) : false,
    });
  }

  return outputs;
}
```

## 実装フェーズ

### Phase 1: FormConfigランタイム（1-2週間）
- `evalFormConfig()` 関数をwidgetに追加
- 既存の `Lambda360Form` が `lambda` だけでなく `FormConfig` も受け取れるようにする
- 既存20 exampleをFormConfigに変換してテスト

### Phase 2: ビルダーUI（2-4週間）
- STEPアップロード → パラメータ設定 → 伸縮設定 → 価格設定 のウィザード形式
- ライブプレビュー（右ペインで常にフォームとモデルを表示）
- FormConfigのJSON保存・読み込み

### Phase 3: 切断面の視覚的設定（2-3週間）
- 3Dモデル上で切断面をドラッグして設定
- バウンディングボックスから自動提案
- 伸縮結果の即時プレビュー

### Phase 4: テンプレートと公開機能（1-2週間）
- 業種別テンプレート（箱型、ブラケット、パネル等）
- 「公開」ボタンで埋め込みコード生成
- FormConfigをAPIで保存・配信

## エスケープハッチ

FormConfigで表現できないケース（複雑なboolean演算、条件分岐ロジック等）のために：

1. **FormConfig内にlambda式を埋め込み可能にする**（将来拡張）
   - `pricing.custom_expr: "width > 500 ? base * 1.2 : base"` のような安全な式評価
2. **「コードモードに切り替え」ボタン**
   - FormConfigからscript.tsのボイラープレートを生成
   - 顧客（または我々が受託で）自由にカスタマイズ
3. **受託対応**
   - ビルダーで8割作り、残り2割をコードで仕上げるハイブリッド運用

## ビジネスインパクト

| | 現状 | ビルダー導入後 |
|---|---|---|
| 導入可能な顧客 | 開発者がいる企業のみ | Web担当者がいれば導入可能 |
| セットアップ時間 | 数時間〜数日（開発） | 30分〜1時間（GUI操作） |
| カバー率 | 100%（コードで何でもできる） | 推定90%（20例中20例をカバー） |
| セルフサービス | 不可 | 可能 |
| サポートコスト | 高（コードレビュー等） | 低（GUI操作の案内のみ） |
