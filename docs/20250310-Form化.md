# Lambda360Form 入出力アーキテクチャ設計

## 1. 概要

Lambda360Form は **宣言的な入出力変換モデル** に基づくコンポーネントである。

ユーザーが定義した **入力部品 (Input)** のパラメータを受け取り、**ラムダ関数 (lambda)** を通じて **出力部品 (Output)** に変換する、という一方向のデータフローを核とする。

```
<Lambda360Form
  input={inputSchema}
  lambda={(input) => output}
/>
```

本文書では、このアーキテクチャにおける入出力の型定義と、レイアウトの改善方針を整理する。

---

## 2. 入力部品 (Input)

`input` は、ユーザーに入力を求めるフォーム要素の宣言的定義である。各部品は `type` で種別を示し、`constraint` で入力の制約を与える。

```typescript
type InputDefinition =
  | UploadInput
  | TextInput
  | NumberInput
  | SelectInput;
```

**Upload** — 画像やPDFなど任意のファイルをアップロードする。サムネイル表示付き。アップロード済みファイルは1年間保持。lambdaには URL文字列が渡される（未アップロード時は `null`）。

```typescript
interface UploadInput {
  type: "upload";
  label: string;
  accept?: string;        // MIMEタイプ制限 (例: "image/*", "application/pdf")
  maxSize?: number;       // 最大ファイルサイズ (bytes)
}
```

**Text** — テキスト入力。`variant` により `<input type="text">` / `<textarea>` / `<input type="email">` を切り替える。lambdaには文字列が渡される。

```typescript
interface TextInput {
  type: "text";
  label: string;
  variant?: "text" | "area" | "email";  // default: "text"
  placeholder?: string;
  default?: string;
}
```

**Number** — 数値入力。`min/max/step` 指定時はスライダー、`enum` 指定時はドロップダウン、制約なしなら数値フィールドとして描画。lambdaには数値が渡される。

```typescript
interface NumberInput {
  type: "number";
  label: string;
  unit?: string;          // 表示用の単位 (例: "mm", "kg")
  default?: number;
  constraint?: { min?: number; max?: number; step?: number }
             | { enum: number[] };
}
```

**Select** — 選択肢から1つを選ぶ。少数ならボタン群、多数ならドロップダウンとして描画。lambdaには選択された `value` 文字列が渡される。

```typescript
interface SelectInput {
  type: "select";
  label: string;
  default?: string;
  options: { value: string; label: string }[];
}
```

**使用例:**

```typescript
const input = {
  logo:   { type: "upload",  label: "ロゴ画像" },
  name:   { type: "text",    label: "会社名", variant: "text" },
  note:   { type: "text",    label: "備考", variant: "area" },
  email:  { type: "text",    label: "メール", variant: "email" },
  width:  { type: "number",  label: "幅", unit: "mm", default: 300, constraint: { min: 200, max: 600, step: 10 } },
  height: { type: "number",  label: "高さ", unit: "mm", default: 150, constraint: { enum: [100, 150, 200] } },
  color:  { type: "select",  label: "色", default: "#cccccc", options: [
    { value: "#cccccc", label: "シルバー" },
    { value: "#336699", label: "ネイビー" },
    { value: "#993933", label: "ワイン" },
  ]},
};
```

## 3. 出力部品 (Output)

`lambda` 関数の戻り値は、ユーザーに何を表示し、どのようなアクションを提供するかを宣言的に定義する。

```typescript
type Output =
  | ShapeOutput
  | BorderOutput
  | MessageOutput
  | ButtonOutput;
```

**Shape** — 3Dモデルビューワー。`ShapeNode` 演算ツリーを受け取り、APIで評価して3D描画する。

```typescript
interface ShapeOutput {
  type: "shape";
  label?: string;
  shape: ShapeNode;
}
```

**Border** — 視覚的な区切り線。セクション間の境界を示す。プロパティなし。

```typescript
interface BorderOutput {
  type: "border";
}
```

**Message** — 条件付きメッセージ。バリデーション結果や注意事項の表示に使う。lambda内の条件分岐で動的に出し分けられる。

```typescript
interface MessageOutput {
  type: "message";
  label: string;
  messageType: "warning" | "error" | "text" | "info";
}
```

**Button** — アクションボタン。メール送信やSlack通知などの外部アクションをトリガーする。

```typescript
interface ButtonOutput {
  type: "button";
  label: string;
  action: "email" | "slack";
}
```

**使用例:**

```typescript
const lambda = (input) => {
  const output = [];

  // 3Dモデル
  output.push({
    type: "shape",
    label: "プレビュー",
    shape: {
      op: "stretch",
      shape: { op: "step", content_hash: "d6cb2eb2..." },
      cut: [100, 100, 75],
      delta: [input.width - 200, 0, input.height - 150],
    },
  });

  output.push({ type: "border" });

  // バリデーション
  if (input.width > 500 && input.height > 180) {
    output.push({
      type: "message",
      label: "幅500mm超かつ高さ180mm超の組み合わせは製造不可です",
      messageType: "error",
    });
  }

  // 参考価格
  output.push({
    type: "message",
    label: `参考価格: ¥${(1500 + input.width * input.height * 0.01).toLocaleString()}`,
    messageType: "info",
  });

  // アクション
  output.push({
    type: "button",
    label: "見積もりをメールで送信",
    action: "email",
  });

  output.push({
    type: "button",
    label: "Slackに通知",
    action: "slack",
  });

  return output;
};
```

## 4. レイアウト改善：縦一列レイアウトへの移行

### 4.1. 現状の課題

現在の `Lambda360Form.tsx` は **左右2カラムレイアウト**（左: 3Dビューワー、右: 入力フォーム）を採用している。

```
┌──────────────────────┬─────────────┐
│                      │  幅:  ███   │
│    3D Viewer         │  奥行: ███  │
│                      │  高さ: ▼    │
│                      │  色:  ●●●   │
│                      │             │
│                      │ [発注へ進む]│
└──────────────────────┴─────────────┘
```

この構成は、デスクトップでは機能するがモバイルでは使いにくく、またGoogleフォームのような「上から順に入力していく」直感的なフローに合わない。

### 4.2. 改善後のレイアウト

**Google Forms のような縦一列レイアウト** に変更する。入力部品と出力部品を上から順に並べ、入力→結果表示→アクションという自然なフローを実現する。

```
┌──────────────────────────────┐
│  幅:   ████████████████  300 │
│  奥行: ████████████████  400 │
│  高さ: [ 100 ▼ ]            │
│  色:   [シルバー] [ネイビー]  │
├──────────────────────────────┤
│                              │
│         3D Viewer            │
│       (Shape出力部品)         │
│                              │
├──────────────────────────────┤
│  ⚠ 幅500mm超は追加料金       │
│  ℹ 参考価格: ¥2,700          │
├──────────────────────────────┤
│  [見積もりをメールで送信]     │
│  [Slackに通知]               │
└──────────────────────────────┘
```

### 4.3. 縦一列レイアウトの利点

- **モバイルフレンドリー**: レスポンシブ対応が不要。最初から単一カラム。
- **Google Forms の馴染み深いUX**: 上から下へ順に入力するフローは誰にとっても直感的。
- **入出力の混在が可能**: 入力部品の間に3Dプレビューやメッセージを差し込める柔軟性が生まれる。
- **組み込み先への適合**: Shopify / WordPress 等、埋め込み幅が限られるケースでも自然に表示される。

## 5. 全体像：Lambda360Form インターフェース

上記をまとめると、`<Lambda360Form>` コンポーネントの新しいインターフェースは以下のようになる。

```typescript
interface Lambda360FormProps {
  input: Record<string, InputDefinition>;            // 入力部品のスキーマ
  lambda: (input: Record<string, any>) => Output[];  // 入力 → 出力の変換関数
}

// 入力部品の型
type InputDefinition =
  | { type: "upload"; label: string }
  | { type: "text";   label: string; variant?: "text" | "area" | "email"; placeholder?: string }
  | { type: "number"; label: string; unit?: string; default?: number; constraint?: NumberConstraint }
  | { type: "select"; label: string; default?: string; options: { value: string; label: string }[] };

// 出力部品の型
type Output =
  | { type: "shape";   label?: string; shape: ShapeNode }
  | { type: "border" }
  | { type: "message"; label: string; messageType: "warning" | "error" | "text" | "info" }
  | { type: "button";  label: string; action: "email" | "slack" };
```

### 5.1. test.html での使用例

```html
<div id="custom-form"></div>

<script src="https://cdn.example.com/lambda360.iife.js"></script>
<script>
  const { initLambda360 } = window.Lambda360;

  initLambda360('#custom-form', {
    input: {
      width:  { type: "number", label: "幅",   unit: "mm", default: 300, constraint: { min: 200, max: 600, step: 10 } },
      height: { type: "number", label: "高さ", unit: "mm", default: 150, constraint: { enum: [100, 150, 200] } },
    },
    lambda: (input) => [
      { type: "shape", label: "プレビュー", shape: {
          op: "stretch",
          shape: { op: "step", content_hash: "d6cb2eb2..." },
          cut: [100, 100, 75],
          delta: [input.width - 200, 0, input.height - 150],
      }},
      { type: "border" },
      { type: "message", label: `参考価格: ¥${(1500 + input.width * 10).toLocaleString()}`, messageType: "info" },
      { type: "button", label: "見積もりをメールで送信", action: "email" },
    ],
  });
</script>
```

## 6. 実装方法

### スキーマの定義場所

入出力部品の型は `openapi/main.tsp` に TypeSpec で定義する。現在 `ShapeNode` が同ファイルで定義されているのと同じ方式に従う。

### 自動生成パイプライン

既存の `make -C openapi generate` パイプラインをそのまま利用する。

```
openapi/main.tsp
    ↓  tsp compile (TypeSpec → OpenAPI)
out/openapi.json
    ↓  @hey-api/openapi-ts
out/client/          ← TypeScript 型 + SDK が自動生成される
```

`InputDefinition`, `Output` などの型は `out/client/` に生成され、`widget/src/Lambda360Form.tsx` から `import type { ... } from '@/out/client'` で利用する（`ShapeNode` と同じパターン）。

### main.tsp に追加する定義（案）

```typespec
// ---------------------------------------------------------------------------
// Input schema (入力部品)
// ---------------------------------------------------------------------------

model UploadInput {
  type: "upload";
  label: string;
  accept?: string;
  maxSize?: int64;
}

model TextInput {
  type: "text";
  label: string;
  variant?: "text" | "area" | "email";
  placeholder?: string;
  default?: string;
}

model NumberConstraintRange {
  min?: float64;
  max?: float64;
  step?: float64;
}

model NumberConstraintEnum {
  enum: float64[];
}

model NumberInput {
  type: "number";
  label: string;
  unit?: string;
  default?: float64;
  constraint?: NumberConstraintRange | NumberConstraintEnum;
}

model SelectOption {
  value: string;
  label: string;
}

model SelectInput {
  type: "select";
  label: string;
  default?: string;
  options: SelectOption[];
}

union InputDefinition {
  upload: UploadInput,
  text: TextInput,
  number: NumberInput,
  select: SelectInput,
}

// ---------------------------------------------------------------------------
// Output schema (出力部品)
// ---------------------------------------------------------------------------

model ShapeOutput {
  type: "shape";
  label?: string;
  shape: ShapeNode;
}

model BorderOutput {
  type: "border";
}

model MessageOutput {
  type: "message";
  label: string;
  messageType: "warning" | "error" | "text" | "info";
}

model ButtonOutput {
  type: "button";
  label: string;
  action: "email" | "slack";
}

union Output {
  shape: ShapeOutput,
  border: BorderOutput,
  message: MessageOutput,
  button: ButtonOutput,
}
```

### フロントエンドでの利用

`Lambda360Form.tsx` では自動生成された型をインポートし、手書きの型定義は持たない。

```typescript
import type { InputDefinition, Output, ShapeNode } from '@/out/client';

export interface Lambda360FormProps {
  input: Record<string, InputDefinition>;
  lambda: (input: Record<string, any>) => Output[];
}
```
