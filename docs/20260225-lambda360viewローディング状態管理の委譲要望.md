# lambda360view 子要素・空状態ハンドリングの委譲 要望書

## 背景と現状の課題

現在、各コンポーネントにおいて「モデルデータのロード中」「モデル未指定（null）」「エラー発生時」などのフォールバックUIを独自に実装して `Lambda360View` の上に被せています。
これらの表示制御に関する責務はビューアコンポーネントである `Lambda360View` 側で一元管理されるべきです。

## 期待する動作・API設計

`model` プロパティが `null` である状態を許容し、さらに画面中央に任意のコンポーネントを表示できる機能の追加を要望します。

### 提案する新しいProps
```tsx
interface Lambda360ViewProps extends Omit<React.HTMLAttributes<HTMLDivElement>, 'children'> {
  // 既存のProps (null を許容するように変更)
  model: ArrayBuffer | null; 
  // ... existing props ...

  // 追加を希望するProps
  /** 画面中央に表示したい任意のReactノード（ローディングスピナーやエラーメッセージなど） */
  center?: React.ReactNode; 
}
```

### 振る舞いの要件
1. **`model` が `null` の場合:**
   モデルを表示しない。
2. **`center` が指定された場合:**
   ビューアの中央に、指定された `ReactNode`（エラー文字列やローディングコンポーネント等）を表示する。`model`の再レンダリング時もシームレスに対応できるよう考慮する。

## メリット
利用者側（`Lambda360Shape` 等）の責務が「データを注入するだけ」にシンプル化され、ボイラープレートコードが削減される。

## 優先度
**中** - 利用側コンポーネントの責務分離とコードの単純化のため。

## 解決状況
**対応完了** - `lambda360view@0.1.11` にて `center` プロパティの追加および `model: null` の許容が反映されました。これに伴い、ウィジェット側の不要なローディング用オーバーレイ実装を削除し、コードを劇的に単純化しました。
