# chijin 改善提案

作成: 2026-03-02
目的: `opencascade-rs` から `chijin` に移行した際に気づいた課題をまとめ、改善を提案する。

---

## 1. `Shape` / `Solid` / `Face` / `Edge` に `Send` 実装がない

### 問題

`chijin::Shape` の内部は `cxx::UniquePtr<ffi::TopoDS_Shape>` だが、cxx の opaque 型は自動的に `Send` にならない。

```
E0277: `*const cxx::void` cannot be sent between threads safely
```

Tokio の `spawn_blocking` のクロージャが `Shape` を返す場合や、axum の `async fn` ハンドラが `Send` バウンドを要求する場合にコンパイルエラーが発生する。

### 影響を受けたコード

`api/src/shape.rs` — `spawn_blocking` で `(sha256, Shape)` を返せなかった（`block_in_place` に変更して回避）。
`api/src/server.rs` — `collect_shape()` が `HashMap<String, Shape>` を返す async 関数で Send エラー。

### 回避策（現状）

`api` 側から依存している `Downloads/chijin/src/lib.rs` に手動で追記：

```rust
unsafe impl Send for Shape {}
unsafe impl Send for Solid {}
unsafe impl Send for Face {}
unsafe impl Send for Edge {}
```

`UniquePtr` による単独所有のため、スレッド間の移動は安全と判断した（`opencascade-rs` 同様の方針）。

### 解決済み（chijin 0.1.1）

`chijin 0.1.1` にて `unsafe impl Send` が公式に実装され、`path` 依存なしに `chijin = "^0.1.1"` で利用可能になった。

~~`chijin` ライブラリ本体で上記 `unsafe impl Send` を提供するか、`send` フィーチャーフラグを追加してほしい。~~

### `Sync` の検討

`Sync` が実装されると `Arc<Shape>` が `Send` になり、**1つの Shape を複数スレッドで同時に読み取る**ことが可能になる。

```rust
// Sync があると Arc<Shape> を複数タスクに渡せる
let shape = Arc::new(shape);
let s1 = shape.clone();
let s2 = shape.clone();
tokio::join!(
    spawn_blocking(move || s1.mesh_with_tolerance(0.1)),  // メッシュ
    spawn_blocking(move || s2.edges().collect::<Vec<_>>()), // エッジ
);
```

ただし **`unsafe impl Sync` は `Send` より慎重な判断が必要**：

- OCC 内部の `Handle<Geom_XXX>` の参照カウントは非アトミックのため、`&Shape` の同時読み取りでもカウント操作がデータ競合を起こす恐れがある
- `mesh_with_tolerance` は内部で `BRepMesh_IncrementalMesh` を構築しキャッシュを書き込む（`&self` でも内部ミュータビリティ）

**推奨**: まず `Send` のみ提供し、`Sync` は OCC のスレッド安全性を検証してから追加する。

## 2. `approximation_segments` の引数が `tolerance: f64` 一本

### 現状

```rust
edge.approximation_segments(tolerance)
```

angular deflection と chord deflection を同じ値で設定している。

### 改善提案

より細かい制御のため、angular/chord を独立して指定できる API も検討してほしい（後方互換のオーバーロードとして）：

```rust
edge.approximation_segments_ex(angular: f64, chord: f64)
```
