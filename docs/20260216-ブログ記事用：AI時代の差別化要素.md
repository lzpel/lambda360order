# プログラマーのみんなは、Claude CodeとAntigravityを使っているか？

すごい時代になったもんだ。
「あれがしたい」「これがしたい」と説明するだけで、コードは自動的に生成されていく。

一つのソフトウェアを作るのに必要なエンジニアの数はどんどん減っていき、もはや一人で会社ができるレベルだろう。

これからのソフトウェアエンジニアは、魔法使いのようなものだと思う。
一人で探求し、AIという杖を使って書かれたソフトウェアを、世界にぶっ放す。

そんな時代に、何が人間の差別化要素になるのか？

私が思うに、それは次の2つだ。

1.  意思（お前は何がしたいの？という部分）
2.  「できること」と「できないこと」の境界線を知っていること

---

## 1. 意思を持つこと

意思は分かりやすい。
そもそも、魔法を使ってやりたいことが無い人に杖を与えても、それはただの鈍器にしかならない。

もちろん自由だ。
だけど、明確な「やりたいこと」がありAIをフル活用する人が魔法を打ち合う世界に、鈍器を持っていくのはどうだろう？
それは、お金や時間を搾取される側になるだけではないだろうか。

## 2. 「できること」と「できないこと」の境界線を知っていること

そしてもう一つ、「できること」と「できないこと」の境界線を知っていることは大事だ。

AIは魔法かもしれないけど、「どこを掘ったら面白い結果が出るのか」までは教えてくれない。

例えば、計算流体力学を知らない小学生が、河川をシミュレーションするプログラムを書けるだろうか？
答えはNoだ。
なぜなら、「そういうことができる」ということを知らないから。

もし、23世紀の人間が現代に来たらどうなるだろう？
たとえ最新鋭のAIじゃなく、Claude Code Opus 4.5とか（未来から見れば）古めのエージェントだったとしても、とんでもないプロダクトを作るかもしれない。

他人は「できない」と思っているが、実は「できる」ということを知っている。
これは大きな武器になる。

ハードウェア、数学、物理、そして様々な分野の総合知。
この知識のレベルこそが、魔法の総合力（出力のデカさ）になるのだ。

---

## 知識人であり、研究者であれ

「できること」と「できないこと」を知っているというのは、単に知識として色々知っている（物知りな）魔法使い、という意味にとどまらない。

高度な仮説検証を回して、誰も知らない知識をぶん回している奴が強いという意味もある。

GoogleやOpenAI、Anthropicが今後も覇者であり続けられるのはなぜか？
それは彼らのR&D部隊という仮説検証工場が、新しい知識を量産し続けているからだ。

AI時代。
それは「強い意思を持つ人」「知識人」、そして「研究者」が、最強の魔法使いになれる時代だと思う。
